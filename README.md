# google-interview-prep

A personal collection of resources and notes for preparing for software engineering interviews, with a focus on algorithmic problem solving.

## Purpose

Interview prep is fragmented across dozens of sites, spreadsheets, and blog posts. This repo is a
single place to track what's worth studying, which problem sets to work through, and what patterns
actually show up in interviews.

## Resources

### Problem sets

- [LeetCode Patterns by seanprashad](https://github.com/seanprashad/leetcode-patterns) — curated
  list of LeetCode problems organized by pattern
- [Top 100 LeetCode questions (Blind list)](https://www.teamblind.com/post/New-Year-Gift---Curated-List-of-Top-100-LeetCode-Questions-to-Save-Your-Time-OaM1orEU) — community-curated high-frequency questions
- [LeetCode tracking spreadsheet](https://docs.google.com/spreadsheets/d/1A2PaQKcdwO_lwxz9bAnxXnIQayCouZP6d-ENrBz_NXc) — progress tracking template

### Patterns reference

- [14 Patterns to Ace Any Coding Interview](https://hackernoon.com/14-patterns-to-ace-any-coding-interview-question-c5bb3357f6ed) — pattern-based taxonomy of interview problems

## Study approach

Work through problems by pattern rather than randomly. Identifying the pattern is half the problem.
Common patterns:

- Sliding window
- Two pointers
- Fast and slow pointers
- Merge intervals
- Cyclic sort
- In-place reversal of linked list
- Tree BFS / DFS
- Two heaps
- Subsets
- Modified binary search
- Top K elements
- K-way merge
- Topological sort
- Dynamic programming

## Setup

No code to install. Clone the repo to keep notes alongside the resource list:

```sh
git clone https://github.com/nadimtuhin/google-interview-prep.git
cd google-interview-prep
```

Add your own notes as markdown files organized by topic or problem category.

```
notes/

  trees.md
  graphs.md
  dynamic-programming.md
  system-design.md
```
